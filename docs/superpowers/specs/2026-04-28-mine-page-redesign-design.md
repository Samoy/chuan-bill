# "我的"页面重构设计文档

## 概述

本设计文档描述"小川记账"应用"我的"页面的重构方案，旨在提升用户体验，遵循市面上应用最佳实践，将"我的"页面打造为个人中心，同时实现账单同步、数据导出等核心功能。

## 设计目标

- **个人中心化**：以用户信息为核心，展示账户状态、个人设置
- **功能整合**：移除设置页面，将所有功能前置到"我的"页面
- **渐进式实现**：先实现基础功能，后续扩展会员、等级等高级功能
- **用户体验优化**：遵循主流应用设计模式，提升操作效率

## 功能模块设计

### 1. 账单同步功能

#### 数据结构变更

本地账单增加同步状态字段：

```typescript
interface BillVO {
  // ... 现有字段
  syncStatus: 'init' | 'success' | 'failed'  // 同步状态
}
```

#### 同步流程

1. 用户点击"数据同步"菜单项
2. 弹出同步状态弹窗，显示：
   - 待同步数量：X条（syncStatus = 'init'）
   - 已同步数量：X条（syncStatus = 'success'）
   - 上次同步时间：YYYY-MM-DD HH:mm
3. 用户点击"开始同步"按钮
4. 前端调用后端批量同步接口
5. 后端返回成功数量
6. 前端根据成功数量，将对应数量的`init`状态账单按时间顺序标记为`success`
7. 弹窗显示同步结果

#### 后端接口改造

修改`POST /api/bill/batchCreate`接口返回值：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "successCount": 5
  }
}
```

### 2. 数据导出功能

#### 导出流程

1. 用户点击"数据导出"菜单项
2. 弹出筛选弹窗（复用`FilterModal`组件）
3. 用户设置筛选条件
4. 点击"导出"按钮，选择格式（Excel/PDF）
5. 显示全局Loading
6. 调用后端导出接口
7. 后端生成文件，返回下载链接
8. 前端触发下载，关闭Loading
9. 如果导出失败，显示错误提示Toast

#### 后端接口

```
POST /api/bill/export
Content-Type: application/json

请求参数：
{
  "startDate": "2026-01-01",
  "endDate": "2026-04-28",
  "type": "expense",           // 可选
  "categoryId": "xxx",         // 可选
  "paymentMethodId": "xxx",    // 可选
  "format": "excel"            // excel 或 pdf
}

响应：
{
  "code": 200,
  "data": {
    "downloadUrl": "https://xxx/file.xlsx"
  }
}
```

### 3. 账号与安全功能

#### 页面结构

合并原有`account.vue`和`password.vue`，新增功能模块：

**修改手机号**
- 显示当前手机号（脱敏）
- 输入新手机号
- 发送验证码
- 验证后修改

**修改密码**
- 输入当前密码
- 输入新密码（带强度指示器）
- 确认新密码

**注销账号**
- 二次确认弹窗
- 输入密码验证
- 后端删除用户数据
- 前端清除本地数据并跳转首页

**登录设备管理**
- 显示当前登录设备列表（设备名称、登录时间、IP地址）
- 当前设备标记为"当前设备"
- 支持单设备下线（点击设备项，二次确认后下线）
- 支持全部设备下线（除当前设备，二次确认后下线）

#### 后端接口

```
PUT /api/user/phone
PUT /api/user/password
DELETE /api/user/account
GET /api/user/devices
DELETE /api/user/devices/{deviceId}
DELETE /api/user/devices
```

### 4. 通知设置功能

#### 通知类型

- **消息推送**：总开关，控制是否接收推送通知
- **账单提醒**：开关 + 提醒时间设置（默认21:00）
- **家庭通知**：开关，控制成员加入/退出、账单变更等通知

#### 实现方式

- 使用`wd-switch`组件控制开关
- 使用`wd-datetime-picker`选择提醒时间
- 设置变更后立即同步到后端
- 关闭消息推送时，账单提醒和家庭通知开关自动关闭

#### 后端接口

```
GET /api/user/notification-settings
PUT /api/user/notification-settings

请求/响应参数：
{
  "pushEnabled": true,
  "billReminderEnabled": true,
  "billReminderTime": "21:00",
  "familyNotificationEnabled": true
}
```

- 活力橙 #FF7D00
- 薄荷绿 #07C160
- 樱花粉 #FF69B4
- 紫罗兰 #8A2BE2

#### 实现方式

- 在"我的"页面显示主题切换入口
- 点击后弹出主题选择弹窗
- 弹窗显示预设颜色选项，当前选中项高亮
- 选择后立即应用主题，弹窗关闭

- 后端实现通知设置接口

**总计预估：6-10天**

## 技术要点

### 前端

- 使用UnoCSS进行样式开发
- 复用现有组件（FilterModal、LoginPopup等）
- 使用Pinia进行状态管理
- 遵循项目现有的代码规范

### 后端

- 利用SaToken实现账号管理功能
- 使用MyBatis-Plus进行数据操作
- 遵循统一的响应格式`Result<T>`
- 使用Lombok简化代码

## 注意事项

1. **数据迁移**：本地账单增加`syncStatus`字段需要考虑数据迁移
2. **错误处理**：同步和导出功能需要完善的错误处理机制
3. **用户体验**：Loading状态、错误提示、成功反馈都需要优化
4. **性能考虑**：大量账单的同步和导出需要考虑性能优化
5. **安全性**：注销账号、密码修改等敏感操作需要二次验证
