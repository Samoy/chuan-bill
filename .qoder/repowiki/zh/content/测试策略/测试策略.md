## 项目结构
小川记账采用前后端分离架构：
- 前端基于 uni-app/Vue3，使用 Alova 进行请求管理，并内置基于 @alova/mock 的 Mock 体系
- 后端基于 Spring Boot 3，使用 MyBatis-Plus、Sa-Token、OpenAPI 等技术栈

```mermaid
graph TB

API --> MOCK["Mock 适配器<br/>@alova/mock"]
API --> HTTP["HTTP 客户端<br/>uniappRequestAdapter"]
FE --> CTRL["后端控制器层<br/>Spring MVC 控制器"]
CTRL --> SVC["服务层接口与实现<br/>IAIService/IUserService/IBillService 等"]
SVC --> DAO["数据访问层<br/>Mapper/XML"]
DAO --> DB["MySQL 数据库"]
CTRL --> EXT["外部服务<br/>百炼 SDK 等"]
```

  - 统一结果包装类与枚举用于断言 code/msg/data 结构

- 数据库集成测试

- 负载测试

  - 将请求速率提升至系统临界点，定位瓶颈（CPU/内存/数据库连接/外部服务）

- 性能基准测试
  - 固定场景与数据集，对比版本间性能变化，建立基线

- 外部服务

## 故障排查指南
- 统一异常处理
  - 全局异常处理器将业务异常与系统异常区分为不同响应结构，便于测试断言
- 业务异常
